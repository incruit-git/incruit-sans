# Incruit Sans 글리프 설계 사양서

이 문서는 **Glyphs 3에서 각 글리프를 다듬을 때 참조하는 메트릭·곡선 가이드**입니다.
`build/generate_glyphs_template.py`가 생성한 기본 템플릿 위에서 아래 사양에 따라 시각적 정제를 진행하세요.

---

## 1. 전체 메트릭 기준

| 항목 | 값 | 설명 |
|------|-----|------|
| **UPM (Units Per Em)** | 1000 | 모든 글자 높이 단위 기준 |
| **Ascender** | 800 | 대문자·위 높이 |
| **Descender** | -200 | 아래로 내려가는 부분 |
| **x-Height** | 500 | 소문자 기본 높이 |
| **Cap Height** | 750 | 대문자 최상단 (Ascender 보다 낮음) |
| **Line Gap** | 200 | 행간 여유 |
| **Post Script Underline Position** | -50 | 언더라인 위치 |
| **Post Script Underline Thickness** | 50 | 언더라인 두께 |

---

## 2. 획(Stroke) 설정

### Regular (weight 400)

| 방향 | 두께 | 설명 |
|------|-----|------|
| **Vertical (세로)** | 80 | H, I, l, ㄴ, ㄱ 등 |
| **Horizontal (가로)** | 75 | E, F 크로스바 등 |
| **Diagonal (대각선)** | 78 | A의 대각선 |
| **Curve (곡선)** | 80–75 | o, a의 측면 (흐름에 따라 변화) |

### Bold (weight 700)

| 방향 | 두께 | 설명 |
|------|-----|------|
| **Vertical** | 140 | Regular의 175% 확대 |
| **Horizontal** | 130 | Regular의 173% 확대 |
| **Diagonal** | 135 | Regular의 173% 확대 |
| **Curve** | 140–130 | 곡선 일관성 유지 |

## 3. 라틴 문자 (A–Z, a–z)

### 대문자 (A–Z)

#### 직선형 (A, H, I, K, L, M, N, T, V, W, X, Y, Z)
- **높이**: 750 (cap-height, ascender보다 50 낮음)
- **너비**:
  - H, I: 좁음 (500–600 UPM)
  - M, W: 넓음 (900–1000 UPM)
  - A, V, X: 중간 (600–700 UPM)
- **베이스라인**: 0 (정렬 필수)
- **곡선 반경** (Min Sans 스타일): 20–30 UPM (모서리 살짝 라운딩)
  - 예: A의 꼭짓점 15–20 라운딩
  - H의 모서리: 직각 유지 (라운딩 최소)

#### 곡선형 (B, C, D, E, F, G, J, O, P, Q, R, S, U)
- **높이**: 750
- **곡선 반지름** (Min Sans):
  - O, D: 완전 원형, 반지름 약 240 UPM (정원성 중요)
  - C, G: 85% 원호, 반지름 240 UPM, 열림각 70° 정도
  - S: 두 개의 S자 곡선, 곡률 일정하게 유지
  - B, P: 위아래 부풀음, 중심선 기준 비대칭
- **대비**: 곡선부의 획 두께 80–75 (흐름)

#### 특수 글자
- **I** (대문자 I):
  - 너비: 300 UPM (세리프 없음, 최소 너비)
  - 높이: 750
  - 중앙선 정렬: ±150
- **J**: 아래로 내려감, descender 영역 진입 (약 -150)
- **Q**: 꼬리 추가, 우측 하단에서 내려옴 (약 -200까지)

---

### 소문자 (a–z)

#### x-height 글자 (a, c, e, m, n, o, r, s, u, v, w, x, z)
- **높이**: 500 (x-height)
- **베이스라인**: 0
- **곡선 반지름** (Min Sans):
  - o, a, e: 반지름 약 150 UPM (부드러운 원형)
  - s: S자 곡선, 작은 반지름 75 UPM, 일정한 곡률
  - c, e: 85–90% 원호 유지
- **너비**:
  - i, l, r, t: 250–350 UPM (좁음)
  - m, w: 700–800 UPM (넓음)
  - a, e, o, u: 450–550 UPM (표준)

#### 어센더 글자 (b, d, f, h, k, l)
- **높이**: 800 (ascender 까지)
- **stem**: 기본 획 두께 80 (Regular)
- **곡선부** (b, d): x-height에서 반지름 150 UPM (o와 일치)
- **f**: 특수형
  - 높이: 800 (ascender)
  - 후크 (hook): 상단 좌측에서 우측으로 휨, 곡률 80–90 UPM
  - 크로스바: x-height 중심 부근

#### 디센더 글자 (g, j, p, q, y)
- **높이**: 500 (x-height, 위 부분)
- **디센더 길이**: -200 (descender 까지)
- **곡선부** (g, p, q): x-height에서의 반지름 150 UPM (o와 일치)
- **g**: 이중 루프
  - 위: x-height 기준 o와 동일 곡선
  - 아래: descender로 내려가면서 작은 루프 형성 (반지름 80 UPM)

## 4. 숫자 (0–9)

### 일반 특성
- **높이**: 750 (cap-height, "lining figures" 스타일)
- **너비**: 모두 일정하게 (Tabular Figures, 정렬용)
- **너비 값**: 550 UPM (각 숫자가 같은 너비를 가짐, 금액·표 정렬 목적)
- **베이스라인**: 0
- **곡선 반지름**: 30–40 UPM (모서리 살짝 라운딩)

### 각 숫자별 메트릭

| 숫자 | 너비 | 곡선 반지름 | 특이사항 |
|------|-----|-----------|--------|
| **0** | 550 | 180 UPM | 완전한 타원 (가로 230, 세로 750) |
| **1** | 550 | 0 | 직선 수직, 좌측 정렬 |
| **2** | 550 | 80–100 | 상단 곡선 + 하단 대각선 |
| **3** | 550 | 80–100 | 이중 S자 곡선, 좌측 열림 |
| **4** | 550 | 0 | 대각선 + 수직 크로스, 직선형 |
| **5** | 550 | 80–90 | 상단 직선 + 하단 곡선 |
| **6** | 550 | 150 | 위 작은 곡선 + 아래 큰 루프 (대칭) |
| **7** | 550 | 0 | 상단 수평 + 대각선 (직선형) |
| **8** | 550 | 100 | 이중 원형, 중심 여백 일정 |
| **9** | 550 | 150 | 6의 상하 반전 |

## 5. 한글 (한글 기본 14자 + 조합)

### 한글 메트릭

| 항목 | 값 | 설명 |
|------|-----|------|
| **높이** | 800 | ascender 높이 (Latin과 동일) |
| **베이스라인** | 0 | Latin과 동일 |
| **기본 너비** | 800 UPM | 1:1 정사각형 (한글 고전 스타일) |

### 기본 14자 (초성 + 중성)

#### 초성 (ㄱ, ㄴ, ㄷ, ㄹ, ㅁ, ㅂ, ㅅ, ㅇ)

| 글자 | 구조 | 곡선 반지름 | 특이사항 |
|------|------|-----------|--------|
| **ㄱ** | ㄱ자 꺾임 | 20 UPM | Min Sans: 우측 상단 살짝 라운딩 (직선형 유지) |
| **ㄴ** | 2개 직선 꺾임 | 30 UPM | 우측 아래 라운딩, Pretendard 계승 |
| **ㄷ** | 개와 유사 | 60 UPM | 우측 곡선, 부드러운 u자형 |
| **ㄹ** | 이중 꺾임 | 20 UPM | 직선 유지, 미세 라운딩만 |
| **ㅁ** | 사다리꼴 | 40 UPM | 상단 열림, 하단 직선 |
| **ㅂ** | 세 부분 | 50 UPM | ㄴ 형태 2개 + 중앙 기둥, 좌측 곡선 |
| **ㅅ** | S자 | 60 UPM | Min Sans 스타일: 부드러운 S곡선, 획 두께 80 |
| **ㅇ** | 원형 | 200 UPM | 완전한 원, 반지름 240 정도 |

#### 중성 (ㅏ, ㅑ, ㅓ, ㅕ, ㅗ)

| 글자 | 구조 | 곡선 반지름 | 특이사항 |
|------|------|-----------|--------|
| **ㅏ** | ㅇ + 막대 | 200 UPM | 원형 좌측, 우측 수직선 |
| **ㅑ** | ㅏ + 곡선 | 80–100 UPM | 하단 곡선 추가, 부드러운 흐름 |
| **ㅓ** | ㅏ 좌우반전 | 200 UPM | 원형 우측, 좌측 수직선 |
| **ㅕ** | ㅓ + 곡선 | 80–100 UPM | 하단 곡선, ㅑ와 대칭 |
| **ㅗ** | 원형 + 막대 | 200 UPM | 완전 원 + 하단 수직선 |

#### 획 두께 (Regular 기준)
- **수직선 (획)**: 80 UPM
- **곡선**: 80–75 UPM (흐름에 따라 변화)
- **특수부**: 75–70 UPM (가는 부분, 세리프 같은 느낌)

## 7. Min Sans 스타일 가이드

### 1단계: 모서리 라운딩
모든 직선 꼭짓점에 **20–40 UPM 라운딩** 추가 (Pretendard와의 차이점)
```
예시: A의 꼭짓점
- Before (Pretendard): 예각
- After (Min Sans): 20 UPM 라운딩으로 부드러움
```

### 2단계: 곡선 부드러움
- 곡선 부분의 **베지어 제어점 위치** 조정으로 flow 개선
  - 가로 흐름: 제어점을 약간 안쪽으로
  - 세로 흐름: 제어점을 바깥쪽으로
- 반지름이 일정한 호 (일정한 곡률)

### 3단계: 획 두께 균형
- **수직 획과 곡선의 비율**: 1.0–1.05 (거의 동일)
- **대각선**: 수직의 97–100%
- 이를 통해 "따뜻한" 무게감 달성

### 4단계: 카운터(내부 공간) 개방도
- 작은 크기(10–12pt)에서도 counter가 흐려지지 않도록
  - 'a', 'e', 'o', 'ㄱ', 'ㄴ' 등의 열림 부분을 넉넉하게
  - x-height 글자의 카운터: 150 UPM 이상 (폐쇄적이지 않음)

## 8. Pretendard 계승 항목

### 메트릭 기반
```
Pretendard 메트릭 + 5% 수평 확장
- 글자 폭: Pretendard 대비 1.05배
- 어센더/디센더: Pretendard와 동일
- 획 두께: Pretendard와 동일
```

### 글리프 구조
다음 글자들은 **Pretendard의 기본 구조를 계승**합니다:
- **Latin**: H, A, E, I, T, O, a, e, i, o, u, n, m
- **한글**: ㄱ, ㄴ, ㄷ, ㄹ, ㅁ, ㅇ (기본 초성)
- **숫자**: 0–9 (메트릭·구조)

단, **Min Sans 라운딩**을 모든 글리프에 일관되게 적용합니다.

### 한글 + Latin 혼용

## 11. 마스터 설정 (Bold 생성)

### Regular (400) → Bold (700) 보간 계획

**Regular** (weight 400) 및 **Bold** (weight 700)를 만든 후, 
Glyphs의 Interpolation 기능으로 중간 weights 자동 생성:

```
Regular (100%)  ────┬──── Medium (375%?)
                     ├──── SemiBold (600%)
                     └──── ExtraBold (800%)
                     
Bold (700%)     ────┴───→
```

**Bold 제작 가이드**:
1. Regular의 모든 글리프를 복사
2. 획 두께만 140 UPM으로 증가 (Regular 80 → Bold 140 = 1.75배)
3. 곡선 반지름은 **정원성 유지**하기 위해 약간 크게 조정 (180 → 240 UPM 등)
4. 너비는 일반적으로 **10–15% 증가** (자간 조정)

## 12. 파일 저장 및 내보내기

### Glyphs 3 작업 흐름

```bash
# 1. 템플릿 생성 (기존)
python build/generate_glyphs_template.py
→ sources/IncruitSans-Regular.glyphs (기본 템플릿)

# 2. Glyphs 3에서 열기
open sources/IncruitSans-Regular.glyphs

# 3. 글리프 수정 (디자이너 작업)
- 각 글리프를 이 사양서 기준으로 조정
- 곡선, 너비, 라운딩 확인

# 4. 저장 및 빌드
python build/build.py
→ fonts/otf/, fonts/ttf/, fonts/webfonts/ 생성
```

### 검증 점검

```bash
# 생성된 폰트 검증
python3 << 'EOF'
from fontTools.ttLib import TTFont
font = TTFont("fonts/otf/IncruitSans-Regular.otf")
print(f"Ascender: {font['hhea'].ascender}")
print(f"Descender: {font['hhea'].descender}")
print(f"Glyph count: {len(font.getGlyphOrder())}")
EOF
```

## 13. 참고: Pretendard vs Min Sans 비교표

| 항목 | Pretendard | Min Sans | Incruit Sans |
|------|-----------|---------|-------------|
| **기본 구조** | 중립적, 기하학적 | 따뜻함, 유기적 | Pretendard 계승 |
| **모서리 라운딩** | 최소 (0–10) | 많음 (40–50) | 중간 (20–30) ✓ |
| **곡선 부드러움** | 표준 | 매우 부드러움 | 부드러움 ✓ |
| **x-height** | 표준 | 표준 | 표준 (500) ✓ |
| **획 대비** | 1.06 | 1.0–1.05 | 1.0–1.05 ✓ |
| **카운터 개방도** | 표준 | 개방적 | 개방적 ✓ |
| **웹 성능** | 경량 | 중간 | 경량 ✓ |
| **용도** | 범용 | 본문 따뜻함 | 이력서 최적화 |
